# Contributing to gategrow

Thanks for helping make learning capture a default, not an afterthought.

## What to contribute

This repo is about **ideas**, not implementations. Good contributions:

- **Translations** — Make the methodology accessible in your language
- **Adaptations** — How you adapted the framework for your domain (research, design, management, teaching)
- **Tool integrations** — Guides for integrating with Notion, Obsidian, Logseq, plain text, etc.
- **Methodology improvements** — Better entry templates, review cadences, quality heuristics
- **Case studies** — A before/after of adopting the framework (what changed?)

Implementation contributions (delivery-gate hooks for other tools) are also welcome — just add them to an `integrations/` directory.

## What NOT to contribute

- The reference implementation lives in [ECC](https://github.com/affaan-m/ECC) — send code fixes there
- Personal growth-log entries — those are yours, not for this repo

## PR Workflow

1. **Fork** the repo
2. **Create a branch** — `git checkout -b your-change`
3. **Write clearly** — One idea per PR. Short paragraphs. Active voice.
4. **Add yourself to the community section** of README.md if you're contributing a significant adaptation or translation
5. **Open a PR** — Describe what you changed and why

## Style

- Markdown files only (`.md`)
- One sentence per line for clean diffs
- Frontmatter for metadata when useful (title, author, date, language)
- Link to sources and related work

## Code of Conduct

Be kind. Be curious. Assume good intent.

This is a space for sharing how we learn, not for arguing about tools or platforms. The ideas are universal — implementations are secondary.
